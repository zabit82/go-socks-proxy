# Go SOCKS5 Proxy

A lightweight and fast SOCKS5 proxy server written in Go. This tool provides a simple proxy tunnel with optional IP whitelisting for basic access control. It supports IPv4, IPv6, and domain name resolutions.

> **Note:** The source code for this application was completely generated by AI.

## Features

- **SOCKS5 Protocol Support**: Fully handles SOCKS5 handshakes. It supports the standard `0x00` method (no authentication).
- **CONNECT Command**: Provides seamless TCP forwarding to IPv4, IPv6, and domain name destinations.
- **IP Allowlist**: Optionally restrict proxy access to specific client IP addresses via environment variables. If left unconfigured, the proxy is open to all incoming connections.
- **Systemd Integration**: Comes with a `Makefile` that simplifies building, installing, and configuring the proxy as a native background system service on Linux.
- **High Performance**: Leverages Go's native concurrent execution and `io.Copy` for robust bidirectional data streaming.

## Prerequisites

- [Go](https://golang.org/dl/) (for building the source).
- `make` utility (optional, for using the provided Makefile commands).
- Linux environment using `systemd` (optional, for automated service installation).

## Building from source

To produce a compiled binary, run:

```bash
make build
```

This compiles the project and strips debug information to keep the file size minimal. It will create an executable file named `socks5-proxy` in the current directory.

## Configuration

You can control the server's behavior using the following environment variables:

| Variable | Default | Description |
| :--- | :--- | :--- |
| `SOCKS_PROXY_PORT` | `1080` | The port the proxy server will listen on. |
| `ALLOWED_IPS` | *(Empty)* | Comma-separated list of allowed client IP addresses. If not set, all IP addresses are allowed. Example: `127.0.0.1, 192.168.1.5` |

## Usage

### Running Manually

You can execute the proxy server directly from your terminal:

```bash
# Allow all connections on port 1080
./socks5-proxy

# Run on a custom port and restrict access to multiple IPs
SOCKS_PROXY_PORT=8080 ALLOWED_IPS="127.0.0.1, 192.168.1.5" ./socks5-proxy
```

### Installing as a Systemd Service (Linux)

You can automatically copy the binary and configure it as a daemon service using the Makefile:

```bash
sudo make systemd
```

This command will:
1. Compile the binary.
2. Install it in `/opt/go-socks-proxy/`.
3. Dynamically generate a systemd service descriptor and place it in `/etc/systemd/system/go-socks-proxy.service`.
4. Reload the systemd daemon, then enable and start the service.

To view the real-time status of the service, run:
```bash
make status
```
*or directly using systemctl:*
```bash
systemctl status go-socks-proxy.service
```

#### Modifying Systemd Settings
If you want to configure the port or the `ALLOWED_IPS` whitelist for the background service, open the service file:
```bash
sudo nano /etc/systemd/system/go-socks-proxy.service
```
Modify the `Environment` lines under the `[Service]` section as needed:
```ini
Environment="SOCKS_PROXY_PORT=1080"
Environment="ALLOWED_IPS=127.0.0.1, 192.168.1.5"
```
Save your changes, then restart the service to apply them:
```bash
sudo systemctl daemon-reload
sudo systemctl restart go-socks-proxy.service
```

### Testing the Proxy

You can quickly verify that the proxy works by routing a `curl` request through it:

```bash
curl -x socks5h://127.0.0.1:1080 https://ifconfig.me
```

## Uninstallation

To easily stop the service, disable it, and remove all associated files (binary and unit files) from your system, run:

```bash
sudo make uninstall
```
